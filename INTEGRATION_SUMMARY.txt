================================================================================
KEY FEATURES
================================================================================

✅ Multi-App Integration
   - TrainLog exercises → Exercise habit
   - Dishrated meals → Meal habits (breakfast, lunch, dinner)
   - Planning completions → Planning habit
   - Gratitude entries → Gratitude habit
   - Manual entries in DDHG

✅ Streak Tracking
   - Per-habit streaks (currently global streak in MVP)
   - Auto-reset after 2 days no completion
   - Streaks persist across app restarts
   - Real-time updates

✅ Plant Growth System
   - 7 growth stages (0=Seed to 6=Flourishing)
   - Based on streak count
   - Automatic stage progression
   - Milestone bonuses at 3, 7, 30 days

✅ DDC (Don't Die Credits)
   - +1 DDC per daily habit completion
   - +3 DDC bonus at 3-day streak
   - +5 DDC bonus at 7-day streak
   - +10 DDC bonus at 30-day streak
   - +3 DDC bonus for perfect day (all 9 habits)
   - Max ~15 DDC per day with bonuses

✅ Real-Time Syncing
   - Firestore listeners for live updates
   - <500ms latency
   - Automatic growth stage transitions
   - No manual refresh needed

================================================================================
GROWTH STAGE PROGRESSION
================================================================================

Stage 0: Seed              (0 day streak)
Stage 1: Rooted            (1 day streak)
Stage 2: Sprout            (2-3 day streak)
Stage 3: Leafing           (4-6 day streak) ← MVP starting point
Stage 4: Budding           (7-10 day streak)
Stage 5: Blooming          (11-15 day streak)
Stage 6: Flourishing       (16+ day streak)

Each stage has:
- Unique plant visual (7 images per plant)
- Scale animation (0.82x to 1.14x)
- Aura glow effect (opacity 30% to 100%)
- Plant metaphor (Lotus, Moonbloom, Iron Fern, etc.)

SECURITY
================================================================================

Firestore Rules:
  ✅ Users can only read/write their own data
  ✅ ddc_ledger writes disabled (server-only)
  ✅ growth_milestones writes disabled (server-only)
  ✅ Auth required for all collections

Google OAuth:
  ✅ Redirect URIs whitelist (localhost + Vercel domain)
  ✅ Client ID for DDHG (separate from other apps)
  ✅ Client secret never exposed in frontend code

Data Privacy:
  ✅ No PII stored except email/name (from Google Auth)
  ✅ No password storage (OAuth only)
  ✅ User can delete account (deletes all Firestore data)
