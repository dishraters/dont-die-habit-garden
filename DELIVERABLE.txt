TECHNICAL DETAILS
==============================================================================

Architecture:
- Each habit app is a standalone Next.js 14 application
- Minimal dependencies (React + Next only)
- Inline CSS styling (no external stylesheets)
- Responsive design (mobile 320px to desktop 1200px+)
- Client-side state management (React hooks)
- localStorage persistence where needed

Webhook Integration:
- All apps POST to: POST /api/heartbeat/complete
- Payload format verified and documented
- Error handling with user feedback
- Async/await pattern used consistently
- Environment variable for DDHG_API (fallback: localhost:3000)

Real-time Updates:
- Dashboard components poll /api/heartbeat/status every 30 seconds
- No WebSocket needed (polling is sufficient)
- All 4 Phase 3 components update automatically

Data Flow:
  User completes habit → POST webhook → Firestore updated → Dashboard polls → UI refreshes
